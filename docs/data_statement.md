## 标注责任链

新批次采用以下流程：

1. 项目作者准备图片、记录来源并排除隐私或许可不清内容。
2. 第一 OCR/VLM 模型生成预标注草稿。
3. 第二模型对照图片复核漏行、错符号、缩进、结构和阅读顺序。
4. 项目作者逐图终审，修订最终 TXT，并决定通过、返工或弃用。
5. 脚本检查配对、空标注、重复、索引和 train/test 哈希重叠。

模型输出不是最终真值。历史批次保证当前文本经过人工检查和最终机械审计，但不补写未保留的模型中间记录。

## 标注规范

- 只转写图片中可见文字，不补全遮挡或画外内容。
- 保留大小写、符号、缩进、换行、路径、表格结构和阅读顺序。
- 不加入解释、总结、Markdown 包装或代码修复。
- 分类模型只用于分组，不决定 OCR 真值。
