# 训练数据构建报告

更新日期：2026-07-10

内部工作流使用采集、预标注、配对审计、重复检查和训练测试泄漏检查脚本。公开仓库保留 `dataset/audit_dataset_quality.py` 作为基础质检入口；训练数据和内部批次目录不公开。

## 采集与合成

- 真实或公开开发场景先进入候选区，不直接进入训练集。
- 自写代码、配置、终端、diff 和多区域内容可在本机渲染，并保留生成脚本和批次记录。
- 拍屏、JPEG 压缩、模糊、低光和小字号等变化用于提高视觉鲁棒性；增强不得改变可见文字真值。
- 新批次在合入前与训练集和测试集执行 SHA-256 去重。

## 预标注与标注规范

- 模型 OCR 结果只作为预标注草稿；VLM 分类结果只用于候选分组，不直接作为 OCR 真值。
- 标注只记录图片中可见文字，保留大小写、符号、缩进、换行、表格结构和阅读顺序。
- 不补全图片外内容，不解释代码，不把代码改写为等价实现。
- 图片与 `.txt` 同名配对，训练索引统一使用 `<image>OCR:` 提示词。

## 人工处理流程

1. 候选采集后执行来源、配对、空标注和重复检查。
2. 模型生成 OCR 草稿或候选分类结果。
3. 项目作者目视筛除错配、不可读、来源不清或非开发场景样本，并修订最终 TXT。
4. 合入前运行 train/test 哈希重叠、批内重复和路径检查。
5. 合入后重新核对 `train.json`、图片和 TXT 数量及引用完整性。
