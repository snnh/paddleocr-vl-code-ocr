# 任务说明

## 任务名称

开发场景代码 OCR。

## 任务定义

输入一张开发场景截图，模型需要输出图片中可见的代码、命令、配置、日志、错误堆栈、文档代码块、API 参数表或相关开发工具文本。输出应尽量保留原始换行、缩进、大小写、标点符号、括号、引号、路径、文件名、数字和阅读顺序。

统一推荐提示词：

```text
<image>OCR:
```

## 目标场景

- IDE 和编辑器中的代码、文件名、标签页、补全框、错误列表。
- Shell、PowerShell、终端命令和命令输出。
- Python/JavaScript/TypeScript/Go/Rust 等常见语言代码。
- YAML、JSON、TOML、INI、环境变量和配置片段。
- Traceback、编译错误、测试失败和诊断日志。
- Git diff、patch、PR 页面和代码 review 片段。
- 文档代码块、Markdown 页面、API 参数表。
- 小字号、暗色主题、压缩、拍屏、遮挡、多区域混排等困难样本。

## 评价重点

本任务不只评价字符相似度，更关注开发者实际可用性：

- 代码符号是否准确，例如括号、引号、冒号、逗号、点号、等号、箭头、管道符和路径分隔符。
- 缩进、对齐、换行和代码块结构是否保留。
- 多区域文本阅读顺序是否合理。
- 是否漏掉主体文本。
- 是否出现解释性文字、Markdown 包装、不可见补全、幻觉或代码改写。
- 复杂结构如 Traceback、Diff、配置层级、表格字段是否可直接读懂。

## 输出约束

- 只输出图片中可见文本。
- 不总结、不解释、不补全不可见内容。
- 不把代码改写成等价代码。
- 不主动添加 Markdown 围栏，除非图片中可见围栏。
- 文件名、标签页、路径与正文之间可用单独一行 `---` 分隔。
