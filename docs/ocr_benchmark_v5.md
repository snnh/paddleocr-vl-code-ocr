# 开发场景 OCR Benchmark v5

更新时间：2026-07-16

Benchmark v5 当前内容版本为 5.2，共 304 题，覆盖 P01-P09 九类开发场景与 simple、medium、hard 三档难度。最终排序使用 `final_score_v5`；NED、全局均值、完成率和安全分只用于诊断。

## 先说结论

模型 v6 使用正式参数 `max_tokens=4096`、`repetition_penalty=1.08`、`temperature=0` 完成 304/304 题，以 `58.4427` 位列短提示组第一。它与 StepFun 的差距为 1.0799 分；考虑 LLM 裁判约 `±0.5` 的展示误差，领先存在但不算悬殊。

模型 v6 真正占优的是稳定性：完成率 99.0132%，安全分 90.7895%，明显高于同组其他通用 VLM。它仍不擅长 P09 多区域混合开发屏和 hard 难度，复杂画面中的区域顺序、漏主体和局部结构错误仍是主要失分来源。

## 评测口径

正式裁判模型为 `tencent/hy3:free`，只写模型名；reasoning 关闭，prompt 版本为 `dev_ocr_judge_v5_compat`。裁判从内容覆盖、符号准确、缩进对齐、结构格式、阅读顺序、噪声与可用性六个维度比较人工真值与模型输出。

积分公式、Severe Badcase 定义和复现命令见 [benchmark_guide_v5.md](benchmark_guide_v5.md)。v4 榜单保留在 [ocr_benchmark_v4.md](ocr_benchmark_v4.md)，仅用于历史回溯。
